# 功能总结（AI作文批阅系统 V2）

## 1. 账号与权限
- 支持账号登录、登出、Token 校验、获取当前用户信息。
- 采用基于 JWT 的认证机制。
- 角色区分为管理员（admin）和学生（student），并在接口与前端路由层做权限控制。

## 2. 管理员端能力
- 作文批量批阅流程：上传作文题目图片、批量上传学生作文图片、启动后台批处理、查询处理进度。
- 学生账号管理：批量导入学生账号、筛选用户列表、重置指定或全部学生密码。
- 批阅记录管理：查看所有记录、按学生查看记录、查看单条记录详情。

## 3. 学生端能力
- 查看“我的批阅记录”（分页查询）。
- 查看批阅详情（仅可访问自己的记录）。
- 查看个人信息页面。

## 4. 智能批阅工作流
- 输入形态：作文题目图 + 学生作文图片。
- 流程特点：OCR 识别、LLM 批阅、后台任务异步执行、实时进度回传。
- 任务结束后自动清理会话临时文件。

## 5. 数据与系统能力
- 数据层使用 SQLAlchemy + SQLite（可扩展）。
- 启动时自动初始化数据库与后台任务管理器。
- 提供统一异常处理、健康检查接口、API 信息接口。

## 6. 前端页面结构
- 管理员页面：控制台、批阅处理、学生管理、批阅记录。
- 学生页面：我的批阅记录、个人信息。
- 提供路由守卫：未登录拦截、角色权限拦截、登录后自动跳转。
